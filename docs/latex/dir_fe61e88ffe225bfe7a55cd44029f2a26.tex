\hypertarget{README.md_autotoc_md2509}{}\doxysubsubsection{\texorpdfstring{Connecting to Mongo\+DB}{Connecting to Mongo\+DB}}\label{README.md_autotoc_md2509}
First, we need to define a connection. If your app uses only one database, you should use {\ttfamily mongoose.\+connect}. If you need to create additional connections, use {\ttfamily mongoose.\+create\+Connection}.

Both {\ttfamily connect} and {\ttfamily create\+Connection} take a {\ttfamily mongodb\+://} URI, or the parameters {\ttfamily host, database, port, options}.


\begin{DoxyCode}{0}
\DoxyCodeLine{await\ mongoose.connect('mongodb://127.0.0.1/my\_database');}

\end{DoxyCode}


Once connected, the {\ttfamily open} event is fired on the {\ttfamily Connection} instance. If you\textquotesingle{}re using {\ttfamily mongoose.\+connect}, the {\ttfamily Connection} is {\ttfamily mongoose.\+connection}. Otherwise, {\ttfamily mongoose.\+create\+Connection} return value is a {\ttfamily Connection}.

{\bfseries{Note\+:}} {\itshape If the local connection fails then try using 127.\+0.\+0.\+1 instead of localhost. Sometimes issues may arise when the local hostname has been changed.}

{\bfseries{Important!}} Mongoose buffers all the commands until it\textquotesingle{}s connected to the database. This means that you don\textquotesingle{}t have to wait until it connects to Mongo\+DB in order to define models, run queries, etc.\hypertarget{README.md_autotoc_md2510}{}\doxysubsubsection{\texorpdfstring{Defining a Model}{Defining a Model}}\label{README.md_autotoc_md2510}

Once we define a model through `mongoose.\+model(\textquotesingle{}Model\+Name', my\+Schema)\`{}, we can access it through the same function


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ MyModel\ =\ mongoose.model('ModelName');}

\end{DoxyCode}


Or just do it all at once


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ MyModel\ =\ mongoose.model('ModelName',\ mySchema);}

\end{DoxyCode}


The first argument is the {\itshape singular} name of the collection your model is for. {\bfseries{Mongoose automatically looks for the {\itshape plural} version of your model name.}} For example, if you use


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ MyModel\ =\ mongoose.model('Ticket',\ mySchema);}

\end{DoxyCode}


Then {\ttfamily My\+Model} will use the {\bfseries{tickets}} collection, not the {\bfseries{ticket}} collection. For more details read the \href{https://mongoosejs.com/docs/api/mongoose.html\#mongoose_Mongoose-model}{\texttt{ model docs}}.

Once we have our model, we can then instantiate it, and save it\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ instance\ =\ new\ MyModel();}
\DoxyCodeLine{instance.my.key\ =\ 'hello';}
\DoxyCodeLine{await\ instance.save();}

\end{DoxyCode}


Or we can find documents from the same collection


\begin{DoxyCode}{0}
\DoxyCodeLine{await\ MyModel.find(\{\});}

\end{DoxyCode}


You can also {\ttfamily find\+One}, {\ttfamily find\+By\+Id}, {\ttfamily update}, etc.


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ instance\ =\ await\ MyModel.findOne(\{\ /*\ ...\ */\ \});}
\DoxyCodeLine{console.log(instance.my.key);\ //\ 'hello'}

\end{DoxyCode}


For more details check out \href{http://mongoosejs.com/docs/queries.html}{\texttt{ the docs}}.

{\bfseries{Important!}} If you opened a separate connection using {\ttfamily mongoose.\+create\+Connection()} but attempt to access the model through `mongoose.\+model(\textquotesingle{}Model\+Name')\`{} it will not work as expected since it is not hooked up to an active db connection. In this case access your model through the connection you created\+:

\DoxyCodeLine{const\ m\ =\ new\ MyModel();}
\DoxyCodeLine{await\ m.save();\ //\ works}

\end{DoxyCode}


vs

\DoxyCodeLine{const\ conn\ =\ mongoose.createConnection('your\ connection\ string');}

\DoxyCodeLine{await\ m.save();\ //\ does\ not\ work\ b/c\ the\ default\ connection\ object\ was\ never\ connected}

For example, this would allow you to broadcast changes about your Documents every time someone {\ttfamily set}s a path in your Document to a new value\+:

\DoxyCodeLine{\ \ this.emit('set',\ path,\ val);}
\DoxyCodeLine{}
\DoxyCodeLine{\ \ //\ Pass\ control\ to\ the\ next\ pre}
\DoxyCodeLine{\ \ next();}
\DoxyCodeLine{\});}

\end{DoxyCode}

\begin{DoxyCode}{0}
\DoxyCodeLine{schema.pre(method,\ function\ firstPre(next,\ methodArg1,\ methodArg2)\ \{}

\DoxyCodeLine{\});}
\DoxyCodeLine{}

\DoxyCodeLine{\ \ //\ =>\ 'altered-\/originalValOfMethodArg1'}
\DoxyCodeLine{}
\DoxyCodeLine{\ \ console.log(methodArg2);}
\DoxyCodeLine{\ \ //\ =>\ 'originalValOfMethodArg2'}

\DoxyCodeLine{\ \ //\ Passing\ no\ arguments\ to\ \`{}next`\ automatically\ passes\ along\ the\ current\ argument\ values}

\hypertarget{README.md_autotoc_md2516}{}\doxysubsubsection{\texorpdfstring{Driver Access}{Driver Access}}\label{README.md_autotoc_md2516}
Mongoose is built on top of the \href{https://github.com/mongodb/node-mongodb-native}{\texttt{ official Mongo\+DB Node.\+js driver}}. Each mongoose model keeps a reference to a \href{http://mongodb.github.io/node-mongodb-native/2.1/api/Collection.html}{\texttt{ native Mongo\+DB driver collection}}. The collection object can be accessed using {\ttfamily Your\+Model.\+collection}. However, using the collection object directly bypasses all mongoose features, including hooks, validation, etc. The one notable exception that {\ttfamily Your\+Model.\+collection} still buffers commands. As such, {\ttfamily Your\+Model.\+collection.\+find()} will {\bfseries{not}} return a cursor.\hypertarget{README.md_autotoc_md2517}{}\doxysubsection{\texorpdfstring{API Docs}{API Docs}}\label{README.md_autotoc_md2517}
