\hypertarget{Readme.md_autotoc_md3097}{}\doxysubsubsection{\texorpdfstring{Parameters}{Parameters}}\label{Readme.md_autotoc_md3097}
Parameters match arbitrary strings in a path by matching up to the end of the segment, or up to any proceeding tokens. They are defined by prefixing a colon to the parameter name ({\ttfamily \+:foo}). Parameter names can use any valid Java\+Script identifier, or be double quoted to use other characters ({\ttfamily \+:"{}param-\/name"{}}).

This means you might be seeing an error when things worked before.\hypertarget{Readme.md_autotoc_md3109}{}\doxysubsubsection{\texorpdfstring{Unexpected {\ttfamily ?} or {\ttfamily +}}{Unexpected {\ttfamily ?} or {\ttfamily +}}}\label{Readme.md_autotoc_md3109}
In past releases, {\ttfamily ?}, {\ttfamily \texorpdfstring{$\ast$}{*}}, and {\ttfamily +} were used to denote optional or repeating parameters. As an alternative, try these\+:


\begin{DoxyItemize}
\item For optional ({\ttfamily ?}), use an empty segment in a group such as {\ttfamily /\+:file\{.\+:ext\}}.
\item For repeating ({\ttfamily +}), only wildcard matching is supported, such as {\ttfamily /\texorpdfstring{$\ast$}{*}path}.
\item For optional repeating ({\ttfamily \texorpdfstring{$\ast$}{*}}), use a group and a wildcard parameter such as {\ttfamily /files\{/\texorpdfstring{$\ast$}{*}path\}}.
\end{DoxyItemize}\hypertarget{Readme.md_autotoc_md3110}{}\doxysubsubsection{\texorpdfstring{Unexpected {\ttfamily (}, {\ttfamily )}, {\ttfamily \mbox{[}}, {\ttfamily \mbox{]}}, etc.}{Unexpected {\ttfamily (}, {\ttfamily )}, {\ttfamily \mbox{[}}, {\ttfamily \mbox{]}}, etc.}}\label{Readme.md_autotoc_md3110}
