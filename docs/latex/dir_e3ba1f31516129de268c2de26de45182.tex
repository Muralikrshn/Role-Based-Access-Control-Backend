\doxysubsection{Detailed Description}


\href{https://nodemon.io/}{\texttt{ }} \hypertarget{README.md_autotoc_md2855}{}\doxysubsection{\texorpdfstring{nodemon}{nodemon}}\label{README.md_autotoc_md2855}
nodemon is a tool that helps develop Node.\+js based applications by automatically restarting the node application when file changes in the directory are detected.

nodemon does {\bfseries{not}} require {\itshape any} additional changes to your code or method of development. nodemon is a replacement wrapper for {\ttfamily node}. To use {\ttfamily nodemon}, replace the word {\ttfamily node} on the command line when executing your script.

\href{https://npmjs.org/package/nodemon}{\texttt{ }} !\mbox{[}Backers on Open Collective\mbox{]}(https\+://opencollective.com/nodemon/backers/badge.svg) !\mbox{[}Sponsors on Open Collective\mbox{]}(https\+://opencollective.com/nodemon/sponsors/badge.svg)\hypertarget{README.md_autotoc_md2856}{}\doxysubsection{\texorpdfstring{Installation}{Installation}}\label{README.md_autotoc_md2856}
Either through cloning with git or by using \href{http://npmjs.org}{\texttt{ npm}} (the recommended way)\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ -\/g\ nodemon\ \#\ or\ using\ yarn:\ yarn\ global\ add\ nodemon}

\end{DoxyCode}


And nodemon will be installed globally to your system path.

You can also install nodemon as a development dependency\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ -\/-\/save-\/dev\ nodemon\ \#\ or\ using\ yarn:\ yarn\ add\ nodemon\ -\/D}

\end{DoxyCode}


With a local installation, nodemon will not be available in your system path or you can\textquotesingle{}t use it directly from the command line. Instead, the local installation of nodemon can be run by calling it from within an npm script (such as {\ttfamily npm start}) or using {\ttfamily npx nodemon}.\hypertarget{README.md_autotoc_md2857}{}\doxysubsection{\texorpdfstring{Usage}{Usage}}\label{README.md_autotoc_md2857}
nodemon wraps your application, so you can pass all the arguments you would normally pass to your app\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{nodemon\ [your\ node\ app]}

\end{DoxyCode}


For CLI options, use the {\ttfamily -\/h} (or {\ttfamily -\/-\/help}) argument\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{nodemon\ -\/h}

\end{DoxyCode}


Using nodemon is simple, if my application accepted a host and port as the arguments, I would start it as so\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{nodemon\ ./server.js\ localhost\ 8080}

\end{DoxyCode}


Any output from this script is prefixed with {\ttfamily \mbox{[}nodemon\mbox{]}}, otherwise all output from your application, errors included, will be echoed out as expected.

You can also pass the {\ttfamily inspect} flag to node through the command line as you would normally\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{nodemon\ -\/-\/inspect\ ./server.js\ 80}

\end{DoxyCode}


If you have a {\ttfamily package.\+json} file for your app, you can omit the main script entirely and nodemon will read the {\ttfamily package.\+json} for the {\ttfamily main} property and use that value as the app (\href{https://github.com/remy/nodemon/issues/14}{\texttt{ ref}}).

nodemon will also search for the {\ttfamily scripts.\+start} property in {\ttfamily package.\+json} (as of nodemon 1.\+1.\+x).

Also check out the \href{https://github.com/remy/nodemon/blob/master/faq.md}{\texttt{ FAQ}} or \href{https://github.com/remy/nodemon/issues}{\texttt{ issues}} for nodemon.\hypertarget{README.md_autotoc_md2858}{}\doxysubsubsection{\texorpdfstring{Automatic re-\/running}{Automatic re-\/running}}\label{README.md_autotoc_md2858}
nodemon was originally written to restart hanging processes such as web servers, but now supports apps that cleanly exit. If your script exits cleanly, nodemon will continue to monitor the directory (or directories) and restart the script if there are any changes.\hypertarget{README.md_autotoc_md2859}{}\doxysubsubsection{\texorpdfstring{Manual restarting}{Manual restarting}}\label{README.md_autotoc_md2859}

nodemon supports local and global configuration files. These are usually named {\ttfamily nodemon.\+json} and can be located in the current working directory or in your home directory. By default, nodemon will only restart when a {\ttfamily .js} Java\+Script file changes. In some cases you will want to ignore some specific files, directories or file patterns, to prevent nodemon from prematurely restarting your application.

This can be done via the command line\+:


\begin{DoxyCode}{0}

In some situations, you may want to wait until a number of files have changed. The timeout before checking for new file changes is 1 second. If you\textquotesingle{}re uploading a number of files and it\textquotesingle{}s taking some number of seconds, this could cause your app to restart multiple times unnecessarily.

\begin{DoxyCode}{0}

\begin{DoxyCode}{0}
\DoxyCodeLine{nodemon\ -\/-\/delay\ 2.5\ server.js}

\begin{DoxyCode}{0}
\DoxyCodeLine{nodemon\ -\/-\/delay\ 2500ms\ server.js}

\end{DoxyCode}


The delay figure is number of seconds (or milliseconds, if specified) to delay before restarting. So nodemon will only restart your app the given number of seconds after the {\itshape last} file change.
