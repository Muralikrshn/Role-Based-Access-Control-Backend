This is a major rewrite of the internals. Externally there are fewer changes\+:


\begin{DoxyItemize}
\item On node\texorpdfstring{$>$}{>}0.\+8 gauge object now prints updates at a fixed rate. This means that when you call {\ttfamily show} it may wate up to {\ttfamily update\+Interval} ms before it actually prints an update. You override this behavior with the {\ttfamily fixed\+Framerate} option.
\item The gauge object now keeps the cursor hidden as long as it\textquotesingle{}s enabled and shown.
\item The constructor\textquotesingle{}s arguments have changed, now it takes a mandatory output stream and an optional options object. The stream no longer needs to be an {\ttfamily ansi}ified stream, although it can be if you want (but we won\textquotesingle{}t make use of its special features).
\item Previously the gauge was disabled by default if {\ttfamily process.\+stdout} wasn\textquotesingle{}t a tty. Now it always defaults to enabled. If you want the previous behavior set the {\ttfamily enabled} option to {\ttfamily process.\+stdout.\+is\+TTY}.
\item The constructor\textquotesingle{}s options have changed– see the docs for details.
\item Themes are entirely different. If you were using a custom theme, or referring to one directly (eg via {\ttfamily Gauge.\+unicode} or {\ttfamily Gauge.\+ascii}) then you\textquotesingle{}ll need to change your code. You can get the equivalent of the latter with\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{var\ themes\ =\ require('gauge/themes')}
\DoxyCodeLine{var\ unicodeTheme\ =\ themes(true,\ true)\ //\ returns\ the\ color\ unicode\ theme\ for\ your\ platform}

\end{DoxyCode}
 The default themes no longer use any ambiguous width characters, so even if you choose to display those as wide your progress bar should still display correctly.
\item Templates are entirely different and if you were using a custom one, you should consult the documentation to learn how to recreate it. If you were using the default, be aware that it has changed and the result looks quite a bit different. 
\end{DoxyItemize}
